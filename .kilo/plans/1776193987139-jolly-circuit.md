# Configure Kilo Agent Manager for Monorepo Worktrees

## Goal
Configure `.kilo/agent-manager.json` for parallel development with git worktrees for backend and frontend agents.

## Current State
- Baseline code committed on `main` branch
- `.kilo/agent-manager.json` has empty `worktrees: {}`
- No agents defined

## Plan

### 1. Update agent-manager.json
Structure:
```json
{
  "worktrees": {
    "backend": {
      "branch": "feature/backend",
      "path": "worktrees/backend",
      "description": "Backend API development"
    },
    "frontend": {
      "branch": "feature/frontend",
      "path": "worktrees/frontend",
      "description": "Frontend React development"
    }
  },
  "agents": [
    {
      "name": "backend-dev",
      "worktree": "backend",
      "description": "Backend API development"
    },
    {
      "name": "frontend-dev",
      "worktree": "frontend",
      "description": "Frontend React development"
    }
  ]
}
```

### 2. Create Worktrees (via bash)
- `git worktree add worktrees/backend -b feature/backend`
- `git worktree add worktrees/frontend -b feature/frontend`

### 3. Install Dependencies
Run `npm install` in each worktree directory.

## Verification
- Each worktree should have its own `/src` and `/server` directories
- Changes in one worktree won't affect the other
- Merge via PRs when features are complete
